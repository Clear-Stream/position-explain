简介

position CSS属性选择用于定位元素的替代规则，使用该属性来完成一些非正常文档流的页面布局，以此来满足特定情况下的各种需求，下面来探究一下其基本用法：

静态定位

position: static; 默认值 默认的布局方式。

由于是默认值，是指元素使用的正常布局行为，不需要特定指出，那么top, right, bottom, left 和 z-index这些属性对其无效，在书写中也就不需要写出。

相对定位

position: relative ; 相对默认的布局位置进行定位（通常会指定给需要定位元素的父级元素 ）。

					.avatar {
						top: 3px;
						left: 7px;
						position: relative;
					}

绝对定位

position: absolute ; 绝对定位元素脱离正常文档流，相对其定位上下文（Positioning Context）进行定位。

指定元素会对其指定的相对定位的父元素发生位置偏移，然后才能确定其最终位置，因此在书写过程中，需要指定top, right, bottom, left 和 z-index值，来为其定位偏移其父元素的具体数值。

定位上下文 Positioning Context

定位上下文就是值绝对定位元素相对哪个元素定位，默认的定位上下文是 。

假如你想设定某个绝对定位元素的定位上下文，则需要在这个元素的某个祖先元素上设置 position: relative。

z-index

z-index由于绝对定位的元素脱离文档流，在一些情况下，比如会有相互遮挡的情况，这时候就需要给定位的元素指定不同的层级进行来显示，而这个属性正是为此，数值越大层级越高。

					.feedback {
						z-index: 1; 
					}

固定定位

position: fixed ; 顾名思义，此属性定位的元素在页面中为固定位置，只对整体页面，即是屏幕窗口负责，指定该属性的元素在屏幕滚动的时候其位置不会发生改变，也是会经常用到的。比如说我们需要在页面的底端增加的返回顶层按钮，以及弹出提示框的背景遮罩层就是用它来完成的。

					.feedback {
						right: 30px;
						bottom: 30px;
						position: fixed;
					}

粘性定位

position: sticky ;  该属性目前由于浏览器的支持性质，很少会用到，暂且不提。

总结

position 通过上述的简单了解，我们知道其共有五种不同的属性值：

  • 1.静态定位默认值 static 正常的而已属性；
  • 2.相对定位 relative 相对默认的布局位置进行定位，不脱离正常的文档流；
  • 3.绝对定位 absolute 脱离正常的文档流，依据父级元素定位；
  • 4.固定定位 fixed 脱离正常的文档流，依据屏幕窗口定位；；
  • 5.粘性定位 sticky 。

根据以上了解，在使用中依据设计及具体情况选定不同的属性值完成页面的编写工作，祝大家工作愉快
